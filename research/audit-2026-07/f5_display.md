# Ф5 — отображение (машинная часть, 2026-07-07)

## Проверено программно (реплика JS-логики рендера в Python)

Прочитаны и проанализированы: `InterlinearLine.vue`, `SongView.vue`, `AnnotationsPanel.vue`, `FootnoteMark.vue`, `utils/ranges.js`, `utils/renderText.js`. Выведены инварианты отображения, все 24 песни прогнаны через проверщик:

1. **Сноски-якоря / goto**: коллизий меток НЕТ (два вида: одна строка+сегмент+слот у локальных; финал спанящей поверх локальной — как П24, там разведено диапазонами). Потерянных якорей нет → «перейти к месту» из панелей работает для всех аннотаций.
2. **continuation_ranges**: у всех спанящих длина = span−1; последний элемент нигде не [-1,-1]/null (иначе метка бы не отрендерилась); span не выходит за границы песни.
3. **Вариант-надстройка (getLineDeParts)**: у всех сегментов с variant_de поле de — подстрока lines_de → стек над DE-строкой рендерится; variant_de/variant_ru всегда парой.
4. **Курсив `*...*`**: пары сбалансированы везде, КРОМЕ намеренной одиночной `*` в П04 stz4L0/ann0 — `*(s)ter-` (астериск реконструированного и.-е. корня). Одиночная звёздочка регекспом не матчится → отображается литерально → ПРАВИЛЬНО. ⚠ Латентная хрупкость: если в эту аннотацию добавить курсив `*слово*`, звёздочки спарятся неправильно.
5. **Ударения U+0301**: все после гласных.
6. **Сборка** `npm run build` ✓; `index.json` — 24 записи, все ready, файлы на месте ✓; title_annotations ≤1 на песню (hover заголовка привязан к первой — не задето) ✓.
