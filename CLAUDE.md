# CLAUDE.md — Winterreise

## Project Overview

Интерактивная веб-страница с подстрочным переводом вокального цикла «Зимний путь» (Winterreise) Вильгельма Мюллера / Франца Шуберта. 24 песни, немецкий оригинал + русский перевод с пословной разметкой и аннотациями.

Хостинг: GitHub Pages. Фреймворк: Vue 3 + Vite.

## Architecture

### Страница (публичный режим)

```
┌──────────┬────────────────────────────────────┐
│ Меню     │  Die Wetterfahne · Флюгер     [☀]  │
│          │                                    │
│ 1. Gute  │  ┌─ Оригинал ──┬─ Перевод ───────┐│
│    Nacht │  │              │                 ││
│ 2. Die   │  │ Der Wind     │ Ветер   играет  ││
│  ►Wetter │  │ spielt mit   │ Der W.  spielt  ││
│    fahne │  │ der Wetter-  │                 ││
│ 3. ...   │  │ fahne        │ с    флюгером   ││
│          │  │              │ mit  der Wetter. ││
│          │  │ Auf meines   │                 ││
│          │  │ schönen      │ На доме моей  ¹ ││
│          │  │ Liebchens    │ Auf  meines     ││
│          │  │ Haus.        │ прекрасной милой││
│          │  │              │ schönen Liebch. ││
│          │  │              │ Haus            ││
│          │  └──────────────┴─────────────────┘│
│ [Поиск]  │  ── Пояснения ──────────────────── │
│          │  ¹ «моей прекрасной милой» — ...    │
└──────────┴────────────────────────────────────┘
```

- **Левая колонка**: немецкий оригинал (строфами, как в стихотворении)
- **Правая колонка**: русский перевод, под каждым русским словом/группой — соответствующее немецкое слово/группа мелким шрифтом
- **Сноски**: нумерованные (`¹`, `²`, `³`...) у отмеченных фрагментов. Нумерация сквозная в пределах песни. Наведение — подсветка текста, который был заранее аннотирован как фрагмент. Клик — всплывающее пояснение
- **Режим пояснений**: внизу страницы все аннотации песни с указанием, к какому месту относятся
- **Меню слева**: 24 песни, немецкие названия, кликабельное
- **Заголовок песни**: немецкое + русское название
- **Поиск**: по всем текстам (оригинал + перевод)
- **Тема**: переключатель светлая/тёмная
- **Печать/PDF**: экспорт всех песен, с аннотациями

### Админ-режим

Доступ по секретному URL: `?admin=<SECRET_KEY>`.

Возможности:
- Редактирование русского текста (inline)
- Редактирование немецкого подстрочника под каждым русским словом (inline)
- Выделение группы русских слов → добавление сноски (номер + пояснение)
- Удаление/редактирование существующих аннотаций
- Сохранение: коммит через GitHub Contents API (токен передаётся в URL или хранится в localStorage)

Два пути редактирования:
1. **Браузер**: админка с inline-редактированием
2. **Claude Code**: правка JSON-файлов данных в CLI

### Data Model

Каждая песня — JSON-файл в `src/data/songs/`:

```json
{
  "number": 2,
  "title_de": "Die Wetterfahne",
  "title_ru": "Флюгер",
  "stanzas": [
    {
      "lines_de": [
        "Der Wind spielt mit der Wetterfahne",
        "Auf meines schönen Liebchens Haus."
      ],
      "lines_ru": [
        {
          "segments": [
            { "ru": "Ветер", "de": "Der Wind" },
            { "ru": "играет", "de": "spielt" },
            { "ru": "с", "de": "mit" },
            { "ru": "флюгером", "de": "der Wetterfahne" }
          ],
          "annotations": []
        },
        {
          "segments": [
            { "ru": "На доме", "de": "Auf... Haus" },
            { "ru": "моей", "de": "meines" },
            { "ru": "прекрасной", "de": "schönen" },
            { "ru": "милой.", "de": "Liebchens" }
          ],
          "annotations": [
            {
              "segment_range": [1, 3],
              "text": "Liebchen — ласковое обращение, буквально «милочка». Герой говорит о возлюбленной, из чьего дома он уходит."
            }
          ]
        }
      ]
    }
  ]
}
```

**Правила данных:**
- `segments` — массив пар {ru, de}. Порядок = порядок слов в русском переводе
- `de` может быть пустой строкой (слово добавлено для русского смысла, нет немецкого аналога)
- `segment_range` — индексы [start, end] включительно, указывают какие сегменты подсвечиваются
- Одна строка `lines_ru` соответствует одной или нескольким строкам `lines_de` (не обязательно 1:1)

- Минималистичный дизайн, без лишних украшений
- Мобильная версия — позже (пока десктоп)

## Content Workflow

1. Находим/создаём перевод песни
2. Размечаем подстрочник (слово↔слово или группа↔группа)
3. Добавляем аннотации к интересным местам
4. Сохраняем JSON (через админку или Claude Code)
5. Push → GitHub Actions → сайт обновляется

## Translation Rules (Правила подстрочного перевода)

Подстрочник — это не литературный перевод, а пословное соответствие для изучающих немецкий.

1. **Каждое немецкое слово должно быть отражено в подстрочнике.** Не пропускай служебные слова (da, schon, doch и т.д.). Каждое слово оригинала должно появиться в поле `de` какого-либо сегмента.

2. **Сохраняй немецкий порядок слов, если русский при этом звучит приемлемо.** Цель — чтобы читатель видел соответствие между строками. Если порядок слов немецкого оригинала даёт понятный русский текст — сохраняй его.

3. **Меняй порядок только когда русский становится нечитаемым.** Например, «На моей прекрасной любимой доме» — так нельзя, переставляем. Но «Тут я подумал уже в своём безумии» — нормально, сохраняем немецкий порядок.

4. **Одно русское слово = одно немецкое слово (по возможности).** Группируй только когда иначе невозможно (отделяемые приставки: pfiff...aus, неразрывные выражения).

5. **`de` может быть пустой строкой** только для слов, добавленных для русского смысла и не имеющих соответствия в оригинале.

6. **Liebchen = любимая** (не «милая»). Lieb- от Liebe (любовь).

7. **Аннотации: всегда показывай текст пользователю перед добавлением в JSON.** Не коммить аннотации без утверждения.

8. **В аннотациях всегда переводи немецкие фрагменты.** Если цитируешь немецкий текст в пояснении, рядом давай русский перевод. Например: «(es eher bemerken — это раньше заметить)», а не просто «(es eher bemerken)».

## Admin Authentication

- URL: `https://<user>.github.io/winterreise/?admin=<SECRET_KEY>`
- SECRET_KEY — произвольная строка, задаётся при первой настройке
- GitHub Personal Access Token — вводится один раз, хранится в localStorage
- Токен нужен с правами `repo` (для записи через Contents API)
